# Geepers Help

Use this as the quick routing page for geepers agents.

## Start Here

If unsure, use one of these first:
- `/team` for major sessions or mixed tasks
- `/scout` for quick project reconnaissance and fast wins
- `/planner` to prioritize TODOs/plans
- `/builder` to implement prioritized work
- `/validator` before deploy or after major changes
- `/swarm` for parallel, broad-spectrum execution across tools/agents

## Orchestrators (Best Default)

- `/geepers_orchestrator_python`: Python projects (CLI/APIs/services)
- `/geepers_orchestrator_web`: Flask web apps
- `/geepers_orchestrator_fullstack`: Node/Express + frontend
- `/geepers_orchestrator_frontend`: UI-only work
- `/quality`: broad quality audit (a11y/perf/api/deps)
- `/geepers_orchestrator_deploy`: deployment + infra coordination
- `/geepers_orchestrator_checkpoint`: end-of-session cleanup/status/snippets

## Common Specialists

- Code/search/refactor: `/geepers_searcher`, `/geepers_scalpel`, `/geepers_refactor`
- Docs: `/readme`, `/geepers_docs`
- Testing/perf/security: `/testing`, `/geepers_perf`, `/geepers_security`
- Data/citations/links: `/doublecheck`, `/publish`, `/geepers_data`, `/geepers_citations`, `/geepers_links`
- Services/infra: `/geepers_services`, `/geepers_caddy`, `/geepers_system_diag`
- Frontend UX: `/geepers_design`, `/geepers_css`, `/geepers_motion`, `/geepers_a11y`, `/geepers_webperf`

## Quick Examples

- "What should I work on?" -> `/planner`
- "Build the next task" -> `/builder`
- "Is this release ready?" -> `/quality`
- "Deploy safely" -> `/geepers_orchestrator_deploy`
- "Need docs now" -> `/readme`
- "Dataset validation/publish" -> `/doublecheck` then `/publish`
- "Need a second-opinion quality pass" -> `/quality`
- "Need two-model critique before building" -> `/dream`
